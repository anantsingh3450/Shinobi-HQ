CORE BOT ARCHITECTURE (VERIFIED WORKING):

ResearchBot
- Input: ResearchQuery (user question text)
- Adapter: DummyResearchSource (mock findings)
- Output: ResearchReport (findings + executive summary)
- Status: ✅ Complete

StrategyBot
- Input: ResearchReport
- Adapter: HeuristicStrategyGenerator (keyword-based rules)
- Output: StrategyProposal (entry/exit rules, confidence)
- Status: ✅ Complete

BacktestBot
- Input: StrategyProposal
- Adapter: HeuristicBacktestEngine (deterministic validation)
- Output: BacktestResult (win_rate, max_drawdown, passed flag)
- Gating: Fails if win_rate < 50 or max_drawdown >= 20
- Status: ✅ Complete

RiskBot
- Input: Account, StrategyProposal, entry_price
- Adapter: CompositeRiskManager (MaxDrawdown, MaxPositionSize rules)
- Output: RiskVerdict (is_approved, reason)
- Gating: Rejects if account would exceed limits
- Status: ✅ Complete

ExecutionBot
- Input: StrategyProposal (approved by Risk)
- Adapter: PaperEngine (simulated fills) + JsonTradeStore (persistence)
- Output: TradeRecord (persisted to trades.jsonl)
- Gating: None (executes after Risk approval)
- Status: ✅ Complete

PortfolioBot
- Input: TradeRecord (executed)
- Adapter: Account model + JsonPortfolioStore (persistence)
- Output: Updated Account (persisted to account_paper.json)
- Status: ✅ Complete
